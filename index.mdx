## How SQLa works

SQLa has two parts: a visual frontend and a secure backend.

<Steps>
  <Step title="Create a session">
    When you open SQLa, you either create a new session or resume an existing one using a UUID. Each session gets its own isolated SQL Server schema so your tables never collide with other users.
  </Step>
  <Step title="Build a query with blocks">
    Use the block toolbox on the left to drag SQL blocks — SELECT, FROM, WHERE, CREATE TABLE, and more — onto the workspace. Connect them together to form a complete SQL statement.
  </Step>
  <Step title="Execute the query">
    Click **Execute** to send the generated SQL to the backend. SQLa parses, validates, and runs the query in your session's schema and displays the results in a table.
  </Step>
  <Step title="Explore the ERD diagram">
    As you create tables with CREATE TABLE blocks, SQLa automatically renders an Entity-Relationship Diagram (ERD) showing your schema visually.
  </Step>
</Steps>
