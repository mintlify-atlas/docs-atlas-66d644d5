SQLa turns SQL learning into a hands-on, visual experience. Instead of typing raw SQL syntax, you drag and drop blocks that represent SQL clauses — SELECT, FROM, WHERE, JOIN, and more — and connect them together to form complete statements. When you execute a query, it runs against a real Microsoft SQL Server database isolated just for your session, so you see actual results rather than simulated output.

## What SQLa is

SQLa is a web application designed for students learning relational databases and SQL for the first time. It removes the friction of syntax errors by letting you compose queries from visual building blocks powered by [Google Blockly](https://developers.google.com/blockly). Each block maps directly to a SQL construct, so you learn the structure and meaning of SQL statements as you assemble them — not after memorizing a reference.

The application has two parts:

- **Frontend** — A Vite-based single-page application with a Blockly canvas where you build queries and a Mermaid-powered ERD diagram that reflects your schema.
- **Backend** — A Node.js + Express API that receives the SQL generated by your blocks, validates it, and executes it against Microsoft SQL Server.

## Key concepts

### Sessions

Every user in SQLa works inside a **session** — a private, isolated environment identified by a UUID. When you create a session, the backend provisions a dedicated SQL Server schema (`session_{uuid}`) that belongs only to you. Tables you create, rows you insert, and queries you run all stay within that schema.

Sessions have no user accounts or passwords. You identify your session by its UUID. Save it so you can resume your work in a future visit.

### Block editor

The block editor is the main workspace. A toolbox panel on the left lists all available SQL blocks organized by category — DDL, SELECT, FROM, WHERE, JOIN, aggregate functions, and more. You drag blocks onto the canvas and snap them together to build a SQL statement. The editor continuously generates the equivalent SQL as you connect blocks.

### SQL execution

When your query is ready, clicking **Execute** sends the generated SQL to the backend. The server parses the statement, checks it against a security allowlist, and runs it in your session schema. Results appear in a table below the editor.

### ERD diagram

As you create tables with DDL blocks, SQLa automatically generates an **Entity-Relationship Diagram** using Mermaid. The diagram updates in real time to show your tables, columns, and relationships — giving you a visual map of the schema you are building.

## Who SQLa is for

SQLa is built for students who are learning SQL and relational database concepts for the first time. No prior SQL knowledge is required. The visual block approach lets you focus on understanding the structure of SQL statements — how clauses fit together, what each keyword means, and how queries produce results — before writing raw SQL.
