---
title: "SQLa quickstart: run your first SQL query in minutes"
description: "Create a session, explore the pre-populated database, build a SELECT query with blocks, and run it against a live SQL Server database in minutes."
---

This guide walks you through opening SQLa, creating your first session, and running a simple `SELECT` query using the block editor. By the end you will have seen blocks generate real SQL and return results from a live database.

<Steps>
  <Step title="Open SQLa in your browser">
    Navigate to the SQLa application URL provided by your instructor or administrator. You will land on the **index page** — the session management screen.

    <Info>
      SQLa runs entirely in the browser. There is nothing to install.
    </Info>
  </Step>

  <Step title="Create a new session">
    On the index page, locate the **"Nueva Sesión"** tab and click it. Then click **Generar** to create your session.

    The application generates a unique session ID — a UUID that looks like this:

    ```text
    3f2a1b4c-8e9d-4f0a-b1c2-d3e4f5a6b7c8
    ```

    This UUID is your session identifier. The backend uses it to create a private SQL Server schema (`session_3f2a1b4c8e9d4f0ab1c2d3e4f5a6b7c8`) that belongs exclusively to you.

    <Note>
      Copy and save your session UUID before continuing. You will need it to resume your work in a future visit. SQLa does not require an account or password — the UUID is your only credential.
    </Note>

    Click **Continuar** to confirm. The modal closes and you are taken to the block editor.
  </Step>

  <Step title="Explore the pre-populated workspace">
    The block editor opens automatically. Every new session starts with a pre-populated `usuarios` table in your schema so you can run queries immediately without creating any tables first.

    The `usuarios` table has this structure:

    ```sql
    CREATE TABLE usuarios (
      id    INT           PRIMARY KEY IDENTITY(1,1),
      nombre NVARCHAR(100),
      edad  INT
    );
    ```

    It already contains two rows:

    | id | nombre | edad |
    |----|--------|------|
    | 1  | Ana    | 22   |
    | 2  | Luis   | 30   |

    You can confirm this by querying the table in the next steps.
  </Step>

  <Step title="Build a SELECT query with blocks">
    In the block editor, you will find the **toolbox** panel on the left side of the screen. Follow these steps to build a `SELECT nombre FROM usuarios` query:

    1. Open the **DML / SELECT** category in the toolbox.
    2. Drag a **SELECT** block onto the canvas.
    3. From the **Expressions** category, drag a **column name** block and type `nombre` as the column value. Connect it to the output slot of the SELECT block.
    4. Open the **FROM** category and drag a **FROM** block onto the canvas. Type `usuarios` as the table name.
    5. Connect the FROM block to the SELECT block — snap it into the clause slot so the two blocks form a complete statement.

    The SQL preview area below the canvas should now show:

    ```sql
    SELECT nombre FROM usuarios
    ```

    <Tip>
      Blocks only connect where they fit — the editor prevents you from attaching incompatible blocks together. If a block does not snap, check that you are connecting it to the correct slot.
    </Tip>
  </Step>

  <Step title="Execute the query">
    Click the **Ejecutar** button (the execute button in the toolbar). SQLa sends the generated SQL to the backend, which runs it in your session's schema and returns the results.

    The output panel below the editor displays the results in a table:

    | nombre |
    |--------|
    | Ana    |
    | Luis   |

    <Note>
      Only the `nombre` column appears because that is the only column you selected. To see all columns, replace the column name block with a `*` (wildcard) expression block.
    </Note>
  </Step>
</Steps>
